# ADR 0001 — SQLite qua Qt SQL (`QSqlDatabase` / `QSQLITE`)

**Trạng thái:** Chấp nhận  
**Ngày:** 2026-05-24

## Bối cảnh

Central Logger lưu cấu hình trạm, catalog cảm biến, time-series và sự kiện trên máy người dùng. Cần chọn lớp truy cập SQLite cho repo Qt 6.11 greenfield.

## Quyết định

- **Engine:** SQLite (file local, mặc định `~/.central-logger/central-logger.db`)
- **API C++:** `QSqlDatabase` với driver **`QSQLITE`**
- **CMake:** `find_package(Qt6 COMPONENTS Sql)` → link `Qt6::Sql`
- **Repository layer:** `QSqlQuery`, transaction qua `QSqlDatabase::transaction()` / `commit()` / `rollback()`
- **Test:** DB in-memory (`:memory:`) hoặc file tạm trong Qt Test

Lớp `Database` (xem [`thiet_ke_db.md`](../thiet_ke_db.md)) là điểm vào duy nhất — mở connection, chạy `001_initial.sql` khi DB mới, expose connection cho repositories.

## Threading

Modbus poll chạy worker thread; **không** dùng chung `QSqlDatabase` connection giữa thread. Ghi DB qua main thread hoặc DB thread riêng (`Qt::QueuedConnection`), hoặc connection riêng per-thread nếu cần sau này.

## Lý do

| Lựa chọn | Đánh giá |
|----------|----------|
| **`QSqlDatabase` + `QSQLITE`** | Khớp MVVM/repository, tích hợp Qt Test, đủ hiệu năng cho poll ~2s và chart 24h; spec [`thiet_ke_db.md`](../thiet_ke_db.md) đã map theo hướng này |
| `libsqlite3` trực tiếp | Thêm wrapper/boilerplate; lợi thế hiệu năng không cần thiết ở quy mô MVP |

## Hệ quả

- Mọi repository mới dùng `QSqlQuery`, không gọi `libsqlite3` trực tiếp.
- Schema mới: [`src/data/db/schema/001_initial.sql`](src/data/db/schema/001_initial.sql); migration: [`src/data/db/migrations/`](src/data/db/migrations/). `PRAGMA user_version` hiện tại = **5**.
- DB cũ hơn app: tự migrate (backup `.bak` trước). DB **mới hơn** app: mở fail + dialog fatal — cập nhật app hoặc xóa file DB.
- Task DB (CMake): bật `Qt6::Sql` trên target app và test targets.
